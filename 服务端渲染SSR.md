服务端渲染SSR

```
    SSR：
        Server side rendering

    即将组件或页面通过服务器生成html字符串，再发送到浏览器，最后将静态标记“混合”为客户端上完全交互的应用程序。
    
    与传统SPA（单页面应用程序）相比，优势：
        更好的SEO，由于搜索引擎爬虫抓取工具可以直接查看完全渲染的页面；
        减少页面首屏加载时间，提高用户的体验
    
    使用注意：
        开发条件限制。
    同时服务端渲染对服务器的压力也相对较大，和服务器简单输出静态文件相比，通过node去渲染出页面再传递给客户开销比较大，
    需要注意准备好相应的服务器负载；



```
