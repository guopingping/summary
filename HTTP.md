HTTP

```
1、HTTP消息
    1xx：信息
    2xx：成功
    3xx：重定向
    4xx：客户端错误
    5xx：服务器错误

2、HTTP方法
    GET、POST

    什么是HTTP？
        超文本传输协议（HTTP）设计目的是保证客户机与服务器之间的通信；
        工作方式：客户机与服务器之间的请求-应答协议
        web浏览器可能是客户端，而计算机上的网络应用程序也可能作为服务器端；

    请求方法：GET POST
        GET：从指定的资源请求数据；
        POST：向指定的资源提交要被处理的数据
    
        GET请求：
            查询字符串(名称/值对)是在GET请求的URL中发送的。

            可被缓存
            参数保留在浏览器历史记录中
            可被收藏为书签
            不应在处理敏感数据时使用
            有长度限制 2048个字符
            只应当用于取回数据

            后退按钮/刷新：无害
            编码类型：
                    application/x-www-form-urlencoded
            只允许ASCII字符
            安全性：较差，发送的数据是URL的一部分。
            可见性：数据在URL中对所有人都是可见的

        
        POST请求：
            查询字符串(名称/值对)是在post请求的HTTP消息主体中发送的。

            不会被缓存
            参数不会保留在浏览器历史记录中
            不能被收藏为书签
            对数据长度没有限制

            后退按钮/刷新：数据会被重新提交
            编码类型：
                    application/x-www-form-urlencoded或multipart/form-data,
                    为二进制数据使用多重编码
            数据类型无限制，也允许二进制数据；
            安全性：比GET安全，参数不会被保留在浏览器历史或web服务器日志中
            可见性：数据不会显示在URL中

3、其他HTTP请求方法
    HEAD：只会返回HTTP报头，不返回文档主体；
    PUT：上传指定的URL方法；
    DELETE：删除指定资源；
    OPTIONS：返回服务器支持的HTTP方法；
    CONNECT：把请求连接转换到透明的TCP/IP通道；
```
